Project Overview
What Is Premier Shop?

Premier Shop is a production-ready, multi-role e-commerce web application built with Laravel 12. It provides a complete retail pipeline — from product browsing and cart management through Stripe-powered checkout, order fulfillment, and driver-assisted delivery — plus a full admin dashboard for store management.

The platform serves three distinct user roles: Customers who shop, Admins/Staff who manage the store, and Drivers who fulfill deliveries. Each role has its own layout, middleware protection, and dedicated controller logic.

Core Capabilities
  • Full product catalog — categories, search, filters, slug-based routing, QR code per product
  • Shopping cart & wishlist — session-backed cart, persistent wishlist, buy-now shortcut
  • Stripe checkout — coupon validation, shipping calculation, tax, order confirmation
  • Order lifecycle — placed → processing → dispatched → delivered / cancelled / returned
  • Driver portal — duty toggle, order queue, proof-of-delivery photo upload
  • Admin dashboard — sales charts, revenue reports, inventory, role-based access control
  • Loyalty rewards — points earned per order, redeemable at checkout
  • Return requests — customer-initiated, item-level, admin-approved workflow
  • Review system — star ratings, text reviews, admin moderation, admin reply
  • QR code scanning — admin scanner finds product by QR, instant stock update

Request Lifecycle

1. Request arrives → Laravel router matches URI against routes/web.php or routes/auth.php.

2. Middleware stack → web group (session, CSRF, cookies) → auth guard → role guards (AdminMiddleware, DriverMiddleware, PermissionMiddleware) → SecurityHeadersMiddleware adds CSP/HSTS headers.

3. Controller action → Business logic executes, queries Eloquent models, calls services (ShippingService), dispatches mail jobs.

4. Response → Blade view rendered with data, or JSON response for AJAX endpoints, or redirect response.

Feature Breakdown — Customer
Shopping
  • Product browse — search by keyword, filter by category, price range, sort by newest/price/popularity
  • Product detail — image, description, price, stock badge, reviews, related products, recently viewed history
  • Cart — add, update quantity, remove, price summary with tax
  • Buy Now — skip cart, go direct to checkout
  • Wishlist — toggle heart icon on product, view full wishlist, move to cart
  • Search suggestions — live AJAX dropdown as you type
Checkout & Payment
  • Address picker — select from saved addresses or enter new
  • Coupon codes — validate and apply percent or fixed discounts
  • Shipping calculator — AJAX-computed fee based on address/weight via ShippingService
  • Loyalty redemption — apply points as discount at checkout
  • Stripe payment — secure PaymentIntent flow
  • Order confirmation — receipt email dispatched, points credited
Orders & Returns
  • Order history — list all past orders with status
  • Order detail — items, shipping address, delivery status, driver info
  • Cancel order — available while status is pending, auto-restock
  • Print invoice — A4 PDF receipt via DomPDF
  • Return request — select items, specify reason, submit for admin review
  • Track return — view status of submitted returns
Profile & Account
  • Profile edit — name, email, avatar, date of birth
  • Address book — multiple addresses, set default
  • Password change — current password verification required
  • Account deletion — password-confirmed, permanent
  • Loyalty rewards — points balance, full transaction history
  • In-app notifications — bell icon with unread count, dropdown list

Authentication & Security
Authentication Flow
  • Registration — email + password, OTP email sent via RegistrationOtp
  • OTP verification — 6-digit code, expiry enforced, view: auth/verify-otp.blade.php
  • Login — rate-limited (5 attempts), remember-me cookie
  • Email verification — signed URL, resend throttled
  • Password reset — token-based, 60-min expiry
  • Confirm password — re-auth gate before account deletion
Middleware Stack
  • AdminMiddleware — checks user->isAdmin() || isStaff(), redirects otherwise
  • DriverMiddleware — checks user->isDriver(), guards all /driver/* routes
  • PermissionMiddleware — checks specific permission slug on role pivot table
  • SecurityHeadersMiddleware — adds CSP, X-Frame-Options, X-Content-Type-Options, HSTS headers

Payments & Shipping
Stripe Integration

Library: stripe/stripe-php v19.4 · Config: config/services.php → STRIPE_KEY / STRIPE_SECRET in .env

Flow: CheckoutController@process creates a PaymentIntent with order total. On success, order record is created, stock decremented, loyalty points credited, OrderReceipt mail dispatched. payment_intent_id stored on order for refund processing.

Refunds: Triggered when admin approves a return request — Stripe refund API called with stored payment_intent_id.

ShippingService — app/Services/ShippingService.php

Injected into CheckoutController via constructor. Calculates shipping fee based on delivery address region, cart weight/volume, and configurable shipping tiers stored in settings table. Called both on checkout page load and via AJAX when address changes.
